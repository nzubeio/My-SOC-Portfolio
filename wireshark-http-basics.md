# Wireshark HTTP Basics

## What I did
Captured my own web traffic using Wireshark and visited a website.

## What I observed

- GET request = My browser is asking the server for a webpage
- Response = The server replied but sent no content (204 No Content)
- What is happening = My browser requested a page and the server responded

## Extra finding

- Host: connectivity-check.ubuntu.com
- This shows the exact server my system connected to

## Key takeaway

I can now see how a browser communicates with a server:
request → response
